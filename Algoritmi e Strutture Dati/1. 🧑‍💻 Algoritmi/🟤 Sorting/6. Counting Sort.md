# Descrizione dell'Algoritmo üìÉ
Counting Sort √® un #algoritmo di ordinamento che **NON si basa sui confronti**.
- Questo algoritmo suppone che ciascuno degli $n$ elementi dell'*array* sia un numero compreso tra 0 e $k$, $k$ √® detto `range dei valori`, si calcola con $k=max(A)-min(A)+1$;
- Il tempo generale √® di $Œ∏(n+k)$;
- Se quindi il range di valori dovesse essere $k=O(n)$, l'algoritmo sar√† eseguito ad un tempo di $Œ∏(n)$, ci√≤ lo rende `molto pi√π efficiente` rispetto agli altri algoritmi che si basano sui confronti;
- Per√≤, se il range dovesse essere molto pi√π grande rispetto ad $n$, allora l'algoritmo sar√† `molto pi√π lento` rispetto agli altri algoritmi che si basano sui confronti (es. $n=2, k=1000000$ -> $Œ∏(1000002)$);
- Ho bisogno dell'array A\[$1 ... n$\] da ordinare e di ***due ulteriori array*** come appoggio:
	- B\[$1 ... n$\]: conterr√† l'output ordinato;
	- C\[$0 ... k$\]: fornisce la memoria temporanea del lavoro;

Counting Sort usa molto spazio per essere eseguito, infatti √® un algoritmo **NON** in loco.

>[!Note]
>- **STABILE**: i valori uguali **mantengono** l'ordine iniziale prestabilito
>- **NON IN LOCO**: ho bisogno di alcune variabili di appoggio (*array*) al variare dell'input


***
L'algoritmo funziona in quattro macro fasi:
1. Azzero l'array C;
2. Scorro A contando in C il numero di occorrenze di ogni elemento di A. Al termine di questa fase C\[$i$\] rappresenta quante volte comprare in A il valore $i$;
3. Per ogni $i$ ($2‚â§i‚â§k$), imposto C\[$i$\] = C\[$i$\] + C\[$i-1$\], cos√¨ da determinare quanti elementi sono minori o uguali ad $i$;
4. Passa tutti gli elementi di A dalla fine verso l'inizio `(in questo modo l'algoritmo √® stabile)` e li posiziono in B sulla base del contenuto di C;

# Pseudocodice üß¨
``` Pseudocodice TI:"Counting_Sort" "FOLD"
void Counting_Sort(A)
	for i = 1 to k:                  // fase 1
		C[i] = 0
	for i = 1 to n:                  // fase 2
		C[A[i]]++
	for i = 2 to k:                  // fase 3
		C[i] = C[i] + C[i-1]
	for i = n down to 1:             // fase 4
		B[C[A[i]]] = A[i]
		C[A[i]]--
```

- #### Tempo di Esecuzione ‚åõ
$T(n) = 2c¬∑k+2c¬∑(k-1)+5c¬∑n$

- Come si pu√≤ facilmente intuire le 4 fasi hanno i seguenti tempi:

> 1. $Œ∏(k)$ $\quad\longmapsto\quad$ si scorre l'array C da $1 ... k$
> 2. $Œ∏(n)$ $\quad\longmapsto\quad$ si scorre l'array A da $1 ... n$
> 3. $Œ∏(k)$ $\quad\longmapsto\quad$ si scorre l'array C da $2 ... k$
> 4. $Œ∏(n)$ $\quad\longmapsto\quad$ si scorre l'array A da $n ... 1$
*** 
- #### Caso Generale ü§î
Come detto in precedenza, Counting Sort non ha casi migliori o peggiori, `il tempo dipender√† soprattutto dal range` $k$ `in relazione con il numero di elementi` $n$.
$T(n) \Rightarrow Œ∏(n+k)$ 

> [!Summary]
> -  ***Caso medio***: $Œ∏(n+k)$

# Codice dell'Algoritmo üêç

```PYTHON TI:"Counting_Sort" "FOLD"
def Counting_Sort(A):
    k = max(A)-min(A)+1
    B = [None] * (len(A))
    C = [0] * k                         # fase 1
    for i in range(0, len(A)):          # fase 2
        C[A[i]] += 1
    for i in range(1, len(C)):          # fase 3
        C[i] += C[i-1]
    for i in range(len(A)-1, -1, -1):  # fase 4
        B[C[A[i]]-1] = A[i]
        C[A[i]] -= 1
```
